# Описание будущей системы (Техническое задание)

**Название проекта:** Student Attendance Tracker (Журнал посещаемости студентов)

**Цель:** Автоматизация учета посещаемости студентов на занятиях с возможностью ручного ввода и автоматической выгрузки данных из Microsoft Teams.

## Основной функционал

### 1. Управление пользователями и ролями

- **Администратор:** управляет пользователями, группами, дисциплинами, настройками системы.
- **Преподаватель:** отмечает посещаемость своих занятий вручную или через импорт из Teams, просматривает статистику по своим группам.
- **Студент:** просматривает только свою посещаемость.
- **Учебная часть (Dean's Office):** просматривает отчеты по любым группам и дисциплинам.

### 2. Управление справочниками

- **Группы:** факультет, курс, номер группы.
- **Дисциплины:** предметы.
- **Студенты:** привязка к группе, email для поиска в Teams, номер студенческого билета.
- **Преподаватели:** привязка к дисциплинам.

### 3. Учет посещаемости

#### Ручной ввод

Преподаватель выбирает дату, группу, дисциплину и отмечает присутствующих и отсутствующих с указанием причины: болезнь, прогул, уважительная причина.

#### Импорт из Microsoft Teams

- Загрузка CSV-файла с отчетом о присутствии из Teams.
- Парсинг файла и автоматическое создание или обновление записей посещаемости.
- Система должна сопоставлять email из Teams с email студента в БД.

### 4. Отчетность

- Формирование статистики посещаемости по студенту: общее количество пропусков, процент посещаемости.
- Формирование статистики по группе и дисциплине за период.
- Экспорт отчетов в Excel/PDF в будущем.

# Структура БД для журнала посещаемости студентов

## Основные таблицы

### 1. `users` — пользователи

- `id`
- `name`
- `email`
- `password`
- `role` (`student/teacher/admin`)
- `group_id` — для студентов
- `teams_email`

### 2. `groups` — учебные группы

- `id`
- `name`
- `course`
- `faculty`

### 3. `disciplines` — предметы

- `id`
- `name`
- `code`

### 4. `group_discipline` — связь групп и предметов

- `group_id`
- `discipline_id`
- `teacher_id`
- `semester`

### 5. `lessons` — занятия

- `id`
- `group_id`
- `discipline_id`
- `teacher_id`
- `date`
- `pair_number`

### 6. `attendance` — посещаемость

- `id`
- `lesson_id`
- `student_id`
- `status` (`present/absent/late/sick`)

### 7. `import_logs` — логи импорта из Teams

- `id`
- `user_id`
- `filename`
- `status`
- `records_processed`

## Главные связи

- Студент belongsTo Группа.
- Группа hasMany Студенты.
- Группа belongsToMany Дисциплины через pivot.
- Занятие belongsTo Группу и Дисциплину.
- Посещаемость belongsTo Занятие и Студент.
